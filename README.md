## How it works

1. **Bootstrap**: SSuKA reads `./kernel/suppertime.md` and builds a bigram graph.
2. **Centers of gravity**: It picks the most connected tokens as "centers" — these are the resonance points.
3. **Historical bias**: Each run creates a tiny `.bin` shard in `bin/`. Future runs blend these shards into the field.
4. **Generation**: When you prompt SSuKA, it walks the bigram graph, biased by historical centers.

**No neural networks. No transformers. Just resonance.**
